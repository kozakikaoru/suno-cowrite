# composing — 作曲ノウハウ参照資料集

このスキルはコマンドではなく、**制作エージェント (songsmith) が作曲工程の前に読む参照資料のコンテナ**です (旧・作曲家 composer の資料。統合後は songsmith の作曲工程が読む)。必読の `02_style-assembly.md` は制作コア束ね `production/references/write_core.md` に連結済みで 1 Read で読めます。条件付きのジャンルプレイブック等はこの読み方ガイドに従って必要時に個別に読みます。手癖で組まず、設計してから Style とセクション演出を決めます。

## 読み方ガイド

### 必読 (毎回)

| 資料 | いつ読む | 内容 |
|---|---|---|
| `references/02_style-assembly.md` | Style を組む前 | レイヤー法、語彙辞典の引き方、抽象要望の翻訳、矛盾検査、Exclude 設計、Persona 差分設計、コンペの差軸 |

### 条件で選択

| 資料 | 読む条件 |
|---|---|
| `references/01_arrangement-craft.md` | セクション演出を作り込む曲、「アレンジが平板」「もっと激しく」系の修正依頼。迷ったら読む |
| `references/03_tiktok-shortform.md` | ショート動画展開の想定・バイラル狙いの曲 |
| `references/04_kpop.md` | K-POP 系の曲 |
| `references/05_vocaloid-net.md` | ボカロ系・ネット発シーンの曲 |
| `references/06_poetry-spoken.md` | 語り主導の曲 (ポエトリー/スポークンワード)。**韻主導のラップなら 10 系へ** |
| `references/07_subcul-indie.md` | インディー/オルタナ系 (shoegaze・dream pop・lo-fi・hyperpop・音響系) |
| `references/08_anison-idol.md` | アニソン的文脈 (キャラソン・タイアップ・TV サイズ)、アイドル曲 |
| `references/09_retro-revival.md` | シティポップ再解釈・昭和歌謡・synthwave / future funk 等のレトロ再解釈 |

プレイブック 03〜09 は段階的に拡充されます。**読む条件に該当する資料がまだ存在しなければ、02 (+必要なら 01) のみで進めてよい**。

06 (ポエトリー) と 04 (K-POP) は dual-audience 資料で、各末尾の「ことば側への申し送り」節は songsmith の作詞工程でも効きます (統合前は作詞家向けの節。songsmith は全節を読んでよい)。

## 設計の流れ (資料の使い所)

1. suno-spec (実効版) で上限・タグ語彙・スライダー定石を確認する (毎回。composer の手順 1)
2. `02_style-assembly.md` のレイヤー法で Style の骨格を組む。抽象要望 (「疾走感」「エモく」等) は 02 §3 の手順で style-vocab 経由の英語化を通す
3. 該当するジャンルプレイブックがあれば主 1 + 副 1 を読み、サウンド署名・構成慣習・Style 完成例を骨格に反映する
4. セクション演出を作り込む曲は `01_arrangement-craft.md` で密度曲線とパラメータ付きタグへの写像を設計する
5. 納品前に 02 §8 と、読んだ各資料末尾のチェックリストを通す

## 運用規則

- ジャンルプレイブックは **主 1 本 + 副 1 本まで**。3 本以上は読まない (混ぜるほど Style が濁り、読量も膨らむ)
- 該当プレイブックの無いジャンル (王道 J-POP バラード等) は **02 (+演出を作り込むなら 01) だけで組む**。無理に近縁プレイブックを当てない
- 語彙辞典 **style-vocab (実効パスは呼び出しプロンプト参照)** は**通読せず、今回の曲に該当するカテゴリだけを拾い読み**する
- 文字数上限・メタタグ語彙・スライダー数値・モデル仕様は、すべて **suno-spec (実効パスは呼び出しプロンプト参照) が正**。本ライブラリ内の数値は設計の目安として扱う
- プレイブックは年単位で古びない骨格だけを扱う。**シーンの最新動向は :trend 調査で補完**し、ブリーフ経由で曲に反映する (2 速度モデル。プレイブックをその場で書き換えない)

## この資料群の前提

- 例 (Style 記述例・進行の類型など) はすべてこの資料のための自作です。実在アーティスト名・実在曲名は本文に書きません (今後の改稿でも追加しません)
- Style 欄・歌詞・タイトルに実在アーティスト名・曲名を書かない規則は、composer のオリジナリティ絶対規則 (第 1 層) に従います
- 修正依頼 (「アレンジが平板」等) の対応手順は 01 §7 の症状別対処表を参照します
