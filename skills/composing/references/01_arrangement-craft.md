# 01 アレンジ craft — 感情を音の設計に翻訳する

ブリーフの感情曲線を「音の総量と質感の変化」に写すための横断技法。Style 欄 (グローバル) とパラメータ付きタグ (セクション単位) のどちらで実現するかまで決めて初めてアレンジ設計は完了する。セクション演出を作り込む曲と、「平板」「盛り上がらない」系の修正依頼で読む。数値上限・タグ語彙・スライダーの正は suno-spec (実効パスは呼び出しプロンプト参照)。

## 1. 感情 → 音の翻訳表

感情の言葉をそのまま Style に書く前に、まず 4 つの物理量 (モード感 / テンポ帯 / 音色群 / ダイナミクス) に分解する。

| 感情の方向 | モード感 | テンポ帯の目安 | 音色群 | ダイナミクス |
|---|---|---|---|---|
| 開けた多幸感 | メジャー | 中速〜速め | 明るいシンセ、ブラス、開いたコーラス | サビで全開 |
| 切なさ・郷愁 | メジャーとマイナーの往復 | 中速 | エレピ、ストリングス、丸いベース | 抑えめから一点で開く |
| 内省・孤独 | マイナー (静的) | 遅め | ピアノ、パッド、小編成 | 小さく保ち谷を深く |
| 疾走・焦燥 | マイナー寄りでも推進重視 | 速い | 刻むギター/シンセ、細かいハイハット | 高止まりで隙間を作る |
| 威圧・ダーク | マイナー、低域偏重 | 遅い〜中速 | 歪んだ低音、金属質パーカッション | 圧を維持し抜き所を 1 点 |
| 高揚ビルド型 | メジャー/マイナー不問 | 一定 | 上昇系シンセ、ライザー的持続音 | 右肩上がり一方向 |

**コード進行の型と感情の対応 (度数表記の類型)**: I–V–vi–IV 系 = 王道の前向きな開け方 / vi–IV–I–V 系 = 切なさと推進の両立 / IV–V–iii–vi 系 = 甘酸っぱさとドラマ性 (J-POP 定番の類型) / ii–V–I 系 = 都会的な洗練 / i–♭VII–♭VI 系 = 悲壮な下降 / **♭VII の借用** = 疾走感の注入 / **同主調の行き来 (メジャー⇔マイナー)** = 場面の明暗切替 / **サビ前の半音〜全音上転調** = 終盤の劇的な持ち上げ。

**実務論 — 進行の直接指定は Suno で効きが不安定 (確度 Med)**: Style 欄に「IV–V–iii–vi で」のような進行指定を書いても安定して反映されない。進行は**その進行が支配的なジャンル・年代・ムードの記述子に翻訳して近似する**のが定石。例: 甘酸っぱいドラマ性 → `j-pop, 2010s, bittersweet, emotional build` / ジャズ的洗練 → `jazzy chords, sophisticated, smooth` / 悲壮な下降 → `dark, melancholic, minor key feel`。度数表記はあくまで「どんな感情の型を狙うか」を自分の中で確定させる設計語として使う。モード感も同様に、`minor key feel` / `modal shifts between major and minor` のようなムード寄りの記述で近似する。

テンポ帯の設計目安: 遅め 60〜90 / 中速 90〜120 / 速め 120〜150 / 高速 150 以上 (音楽一般の慣習値。ジャンル固有の帯は各プレイブック、Style 欄での表記と効かせ方は suno-spec §5 が正)。BPM 数値の単独指定は効きが弱く、**エネルギー語との併記**が安定する (確度 Low〜Med)。

## 2. 音の抜き差し — セクション密度曲線

アレンジの単位は「セクションごとの密度 (音数 × 帯域の広さ × レイヤー数)」。ブリーフの感情曲線 (セクション × 強度) を密度曲線に写してから、増減の手段を選ぶ。

- **翻訳規則**: 感情強度が上がる = 密度を足す、より効くのは**直前で抜く**。強度の谷 = 低域か上物のどちらかを意図的に空ける
- **単調の禁止**: 同じ密度が 2 セクション続いたら変化点を作る (1 楽器の出入りでよい)
- **ビルド**: 反復を細かく、帯域を上へ、持続音を重ねる。`[Build]` 区間は歌詞を薄くして音に譲る
- **落ちサビ**: 最後のサビの前に要素を大きく抜いた静のセクションを置き、直後の最大サビを対比で立てる
- **アンチドロップ**: サビ頭であえてドラムか上物を抜き、フックと低音だけ残す。2 周目で全開に戻す (ビルドで期待を最大化しておくのが前提)
- **静→爆発**: 曲頭を最小編成で始め、1 点で全開にする。振れ幅そのものが感情表現になる
- **頭 5 秒とフック区間**: 冒頭 5 秒以内に曲の署名になる音を置き、単体で成立する 15 秒程度のフック区間を曲中に 1 箇所設計する (ストリーミング/ショート動画時代の横断慣習。バイラル特化の詳細は 03 プレイブック)

## 3. パラメータ付きタグへの写像

密度曲線をセクション単位で実現する主手段が `[Bridge: stripped down, piano only]` 形式のパラメータ付きタグ (歌詞欄)。**タグ本体の語彙は suno-spec §4 の一覧にあるものだけ**を使い、修飾子は Style 欄と同系の英語記述子で書く。

| 密度曲線の意図 | 写像例 |
|---|---|
| 谷を作る (落ちサビ前) | `[Bridge: stripped down, vocals and piano only]` |
| 右肩上がりの助走 | `[Pre-Chorus: building intensity, rising synth]` |
| 最大化 (大サビ) | `[Chorus: full band, soaring vocals, wall of sound]` |
| 圧を残して閉じる | `[Outro: fade out, echoing vocals]` |

- グローバル Style と矛盾する指示を入れない (アコースティック宣言の曲に `heavy distortion` のタグ等)
- **全セクションに付けない**。感情曲線の山谷が動く場所だけに絞る (全部に付けると対比が消える)

## 4. ボーカルディレクション

声は最大の感情装置。Style 欄の記述子は次の 3 系統から選び、合計 1〜2 語 + 演技 1 語までに抑える。

- **温度**: warm / airy / breathy / sweet ⇔ cold / deadpan / detached
- **力感**: powerful belt / soaring ⇔ soft / hushed / whisper-like
- **距離**: intimate close-mic (近い・私的) ⇔ distant, reverb-washed (遠い・風景的)
- **演技**: confident / vulnerable / raw emotional delivery / restrained (抑制はサビの解放を立てる)

コーラスワークは Style 側の記述 (`layered harmonies` / `stacked vocals` / `gang vocals` / `call-and-response chants`) と歌詞欄タグ (`[Harmony]` `[Choir]` `[Backing Vocals]` `[Duet]` `[Ad-lib]` — 語彙は suno-spec §4) を併用する。サビだけハモを厚くする指定はパラメータ付きタグ (`[Chorus: layered harmonies]`) が確実。掛け合いは `[Duet]` + 男女の割当を書くが、Vocal Gender 設定 (主ボーカル) と矛盾させない。

## 5. 低音とリズムの土台

hiphop の低音ルール (songwriting/references/10 §5) はジャンル一般に翻訳できる。土台が書かれていない Style は「薄い」事故の最大原因。

1. **低音の記述を省かない** — ベースの性格を必ず 1 語書く: `deep sub bass` (沈む持続) / `warm round bass` (丸く支える) / `driving bassline` (推進) / `funky slap bass` (跳ねる)
2. **低音は 1 系統に絞る** — 電子系サブベースと生ベースの併記のような二重低音は低域が濁る。どちらが主かを決めて 1 つだけ書く
3. **細かい刻みが質感の半分** — ハイハット/リズムの解像度を書く: `rolling hi-hats` (細分・現代的) / `crisp hi-hats` (歯切れ) / `swung drums` (揺れ) / `four-on-the-floor` (踊れる等拍)
4. **小さいスピーカー対策は倍音で** — スマホ試聴で低音が消える症状には、低音を「増やす」のではなく `warm saturated bass` 等の倍音・質感語で中域に写す
5. **土台から書く** — Style の楽器枠は低音 → ドラム → 上物の順で決める。上物から書くと土台を書き忘れる

リズム型と体感の対応: four-on-the-floor = 踊れる推進 / バックビート強調 (`punchy drums`) = ロック的安定 / スウィング/シャッフル = 余裕と揺れ / ハーフタイム = 重さ・貫禄 / シンコペーション (`syncopated groove`) = 前のめり。

## 6. プロダクション質感と年代

- **年代記述子は最強クラスの 1 語** — `80s` `90s` `2010s` 等はその時代の録音・処理・音色の慣習一式を引き寄せる。年代は 1 つに絞る (二重指定は方向が濁る。レトロ再解釈の意図的な混成は 09 プレイブック)
- **仕上げの軸**: clean / polished / glossy (商業的な艶) ⇔ raw / gritty / lo-fi (宅録感・体温)。`analog warmth` `tape saturation` は温かさの定番、`vinyl crackle` `tape hiss` はノイズをあえて演出に使う
- **空間の軸**: dry, upfront (声が近く言葉が立つ — 高速曲・語り物向き) ⇔ spacious, reverb-heavy (風景として聴かせる — ドリーミー系向き)。**空間の軸は 1 方向に決める**。dry と reverb-heavy の併記は矛盾タグの代表例
- 質感語は「エフェクトの選択」に翻訳される: warm → アナログ系飽和 / gritty → 歪み / spacious → 深いリバーブ / intimate → クローズマイク・小編成 / cinematic → オーケストラルな広がり

## 7. 修正依頼の症状別対処表

症状の解消に必要な層だけを動かす (全面書き換えは別の事故を生む)。変更は 1 回につき 1〜2 要素まで。

| 症状 (P の言葉) | 疑う層 | 対処 |
|---|---|---|
| 「平板」「のっぺりする」 | §2 密度曲線 | セクション対比を作る: 谷を 1 つ掘る (パラメータ付きタグで抜く) + `dynamic contrast` 系の語 |
| 「盛り上がらない」 | §2 ビルド不在 | `[Pre-Chorus: building]` を足す / サビ直前に抜き / サビへ `soaring` `explosive` |
| 「うるさい」「ごちゃごちゃ」 | Style の積みすぎ | 楽器枠を 2〜3 に減らす / 密度系の語 (maximalist 等) を外す / 帯域の主役を 1 つに |
| 「軽い」「薄い」 | §5 土台 | 低音の性格語が抜けていないか。`deep sub bass` 等を追加し、低音は 1 系統に |
| 「歌が埋もれる」 | §4 + §6 | `upfront vocals` へ / リバーブ系の語を弱める / 上物の楽器を 1 つ削って帯域を空ける |
| 「古くさい」「今っぽくない」 | §6 年代 | 年代タグの見直し (無指定は既定で現代寄り、意図しないレトロ化は年代語の混入を疑う) |
| 「激しくして」 | 多層 | どの層か切り分けてから: テンポ帯 / 密度 / 歪み (gritty, distorted) / ボーカル力感 / ドラムの強さ |
| 「なんか違う (曖昧)」 | ブリーフ | 音の問題と決めつけない。ブリーフの感情曲線と Style の対応を見直し、マネージャー経由で症状の具体化を求める |

## 8. 推敲チェックリスト

- [ ] ブリーフの感情曲線の山と谷が、密度曲線とタグ写像に現れている (山谷のないベタ塗りになっていない)
- [ ] 同じ密度が 2 セクション以上続いていない (続くなら意図がある)
- [ ] 低音の性格とリズムの解像度が Style に書かれており、低音は 1 系統
- [ ] ボーカル記述が 3 系統から 1〜2 語 + 演技 1 語以内で、Vocal Gender 設定と矛盾しない
- [ ] パラメータ付きタグはタグ本体が suno-spec §4 の語彙内で、グローバル Style と矛盾せず、山谷の場所だけに付いている
- [ ] 空間の軸 (dry ⇔ spacious) と年代が 1 方向に決まっている
- [ ] 文字数上限・タグ数・スライダー値は suno-spec (実効版) で確認した
