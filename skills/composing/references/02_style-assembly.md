# 02 Style 組み立て — 記述子を層で設計する

Style 欄を「思いついたタグの列挙」ではなく「役割の決まった層の集合」として組むための必読資料。語彙辞典 **style-vocab (実効パスは呼び出しプロンプト参照)** と対で使う。並び順・タグ数・文字数上限などの基本規約と正確な数値は suno-spec §5 (実効版) が正で、本資料はその上に載る設計手順を扱う。

## 1. レイヤー法 — 基本形の語数配分

Style は次の層で組む。**合計はフル版で 10 タグ前後** (上限運用は suno-spec §5)。

| 層 | 語数 | 役割 | 例 (自作) |
|---|---|---|---|
| ジャンル | 1 (サブジャンルまで特定して 1 つ) | 最重要。**必ず先頭** | `japanese city pop` |
| 年代/文脈 | 1 | 録音・処理の慣習一式を引き寄せる | `80s` |
| ムード | 2 | 感情の方向。generic 語 (happy/sad) を避け特異的な語に | `nostalgic, bittersweet` |
| 楽器 | 2〜3 | 低音 → ドラム → 上物の順で決める (01 §5) | `funky slap bass, shimmering electric piano` |
| ボーカル | 1〜2 | 温度/力感/距離 (01 §4) | `smooth female vocals` |
| プロダクション | 1〜2 | 仕上げと空間の軸 (01 §6) | `analog tape warmth` |
| 非音楽語 (情景/エネルギー) | 1〜2 | 場面・動きの複合ショートカット (§3) | `night drive feel` |

- **先頭の語ほど強く効く** (プロンプトは高次属性に圧縮され、先頭・高頻度語が支配的 — 確度 Med)。飾りを先頭に置かない
- 全層を埋める義務はない。**枠が余っても足さない** (語数を盛るほど平均化して個性が消える)
- テンポを書くなら BPM 単独ではなくエネルギー語との併用が安定 (表記は suno-spec §5 の `bpm feel` 形式)

## 2. 語彙辞典 (style-vocab) の引き方

- **通読しない**。今回の曲で埋めたい枠 (ムード / 情景 / 質感 / エネルギー / 日本文化語) に該当するカテゴリだけ拾い読みする
- **確度 High / Med の語を骨格に使う**。辞典の確度列がそのまま採用基準になる
- **確度 Low・未検証の語は「実験枠」1 語まで**。使ったら【判断メモ】に「実験枠: 〇〇 (確度 Low)」と明記し、P が結果で検証できるようにする (実測結果は update-spec 経由で辞典に還元される)
- 辞典に無い語を発明したときも実験枠として数え、判断メモに書く (次回調査の候補になる)
- **日本語の意図語は英語に訳して使うのが原則** (日本語直書きは認識が不安定)。訳し方は §3 の手順で
- 辞典の「効かない・危険語」カテゴリは Style を確定する前に一度照合する

## 3. 抽象要望の翻訳 — 「疾走感」「エモく」を英語記述子に落とす

P・ブリーフの日本語の意図語を英語 Style に変換する標準手順。勘で英訳せず辞典を経由する。

1. **抜き出す** — ブリーフ・P の発言から意図語を拾う (「疾走感」「エモく」「チルに」「夜っぽく」)
2. **分解する** — その語が指すものを 4 枠に仕分ける: 感情 (ムード) / 動き・速度 (エネルギー) / 場面 (情景) / 手触り (質感)。「エモく」のような多義語は 2 枠以上に割れる
3. **辞典で置換する** — style-vocab の該当カテゴリから確度 High/Med の語を選ぶ。1 意図語 = 1〜2 記述子まで
4. **矛盾検査に回す** — 既存の層 (ジャンル・楽器・年代) と §4 で照合する
5. **対応を記録する** — 【判断メモ】に「疾走感 → driving (辞典 Med)」の形で 1 行残す (P が聴感で答え合わせできる)

変換例 (語の最終確度は辞典側が正):

| 意図語 | 分解 | 変換例 |
|---|---|---|
| 疾走感 | 動き | `driving` を第一候補に、`fast-paced` やテンポ帯を併記 |
| エモく | 感情 + 動き | `bittersweet` + `soaring` など分解して 2 語 (generic な `emotional` 単独にしない) |
| チルに | 動き + 質感 | `laid-back, mellow` (+ 場面を足すなら情景 1 フレーズ) |
| 夜っぽく | 場面 | `night drive` 系の情景フレーズ 1 つ |
| 切なく | 感情 | `melancholic` または `wistful` (静的なら後者) |
| 派手に | 密度 | `maximalist, glossy production` |

- **情景フレーズは 1 曲 1 つまで**。ジャンル等の主柱を立てた後ろに置く (情景だけの長文は圧縮で切り捨てられる)
- 翻訳しても方向が曖昧な要望 (「いい感じに」) は、翻訳せずマネージャー経由で具体化を求める

## 4. 相性と矛盾の検査

組み上がった Style を確定する前に、競合タグの類型で総当たりする。

| 矛盾の類型 | 例 | 扱い |
|---|---|---|
| 空間 | `reverb-heavy` × `lo-fi` / `dry` | どちらかに決める (01 §6) |
| 温度 | `warm` × `cold, icy` | どちらかに決める |
| エネルギー | `calm` × `aggressive` | 原則同居禁止。意図的な緊張ペアは `warm but lonely` のように **but で 1 フレーズ化して 1 枠扱い** (確度 Low〜Med、実験枠として判断メモに書く) |
| 密度 | `minimal, sparse` × `lush, maximalist` | どちらかに決める |
| 年代 | `80s` × `modern` | 年代は 1 つ (意図的な再解釈は 09 プレイブック) |
| ジャンル多重 | サブジャンル 3 つ以上の併記 | 主 1 + 修飾 1 まで。それ以上は平均化する |

- **タグ数と Style Influence は連動して決める**: タグを絞るほど追従度を高くできる (高追従 × 少数タグが定石。数値の定石帯は suno-spec §3 が正)。タグ 10 超のまま高追従にすると競合して濁る
- 検査手順: 各層を 1 回ずつ読み直し、(a) 同じ枠に 3 語以上入っていないか、(b) 上の類型に当たるペアがないか、(c) 歌詞の内容・物語が混入していないか (Style は音響指示専用)、を見る

## 5. Exclude 設計の定石

否定は Style 欄に書かず Exclude Styles へ (Style 欄の否定形は不安定 — suno-spec §5)。

- **少数精鋭**: 「この曲で混入すると事故る要素」だけを入れる (5 項目以内目安)。心配性で並べると生成の幅を不必要に削る
- **「事故ったら足す」運用**: 初回から埋め尽くさず、テイクで混入した要素を修正時に追加するのが基本
- 定石の類型: ジャンル汚染防止 (バラードに `edm, dubstep`) / 楽器事故 (アコースティック曲に `heavy distortion`) / ボーカル事故 (インスト曲に `vocals` — トグル・タグとの三重指定の一部) / ムード反転防止 (ダーク曲に `upbeat, cheerful`)
- Style 側と同じ語を両方に書かない (Style に `acoustic` + Exclude に `electronic` は可。Style の語を Exclude に入れるのは矛盾)

## 6. Persona 適用時の差分設計

前提 (suno-spec §6): Persona = アーティストの同一性の錨 / Style 欄 = 曲ごとのプロデューサー指示。登録済みアーティストでは 2 変種 (適用時 / フル) を納品する。

- **適用時 Style = フル版からアイデンティティ層を引き算したもの**: ジャンル帯・基準ボーカル像 (artist.yaml の genre_band / vocal 由来) は Persona が担うため書かない。残すのは曲固有の層 = ムード / 情景 / セクション演出寄りの楽器追加 / 今回だけの質感 (語数の目安は suno-spec §6 と composer 定義に従う)
- **検査 2 点**: (a) 適用時 Style に genre_band / vocal 由来の語が残っていないか (残すと Persona と競合して崩れる)、(b) フル版と適用時版の曲固有層が同じ方向を向いているか (2 変種は「同じ曲の 2 通りの入稿」であって別の曲にしない)
- 基準ジャンル帯から遠い曲は Persona 自体が崩れやすい — 適用可否の判断枠組みは composer 定義 (agents/composer.md) の規則に従う

## 7. コンペ 2 案の差軸カタログ

コンペモード (デモ A / B) では、**差は 1 軸だけで作る**。2 軸以上動かすと P がどの違いを聴いているのか分からなくなる。どの軸でもブリーフのテーマと genre_band の範囲内に収める。

| 差軸 | A 案 ⇔ B 案の例 |
|---|---|
| 年代軸 | 同じ曲想を `80s` 解釈 ⇔ `modern` 解釈 |
| 密度軸 | `sparse, intimate` (小編成) ⇔ `lush, layered` (壁) |
| ボーカル温度軸 | `breathy, intimate vocals` ⇔ `powerful soaring vocals` |
| 情景軸 | 同じムードを別の場面語で (`night drive` ⇔ `rainy day cafe`) |
| 動力軸 | 生バンド感 (`live drums, organic`) ⇔ 打ち込み (`electronic beat, synth-driven`) |

【コンペ提案】の冒頭に「今回の差軸: 〇〇軸」を 1 行で書き、各案の狙いを対にして示す。

## 8. 推敲チェックリスト

- [ ] 層の配分が §1 の基本形に収まっている (フル 10 タグ前後 / Persona 適用時は曲固有層のみ)
- [ ] ジャンルが先頭にあり、サブジャンルまで特定されている
- [ ] 非音楽語は確度 High/Med から 1〜2 語。Low・未検証・自作語は実験枠 1 語まで + 判断メモに明記した
- [ ] 情景フレーズは 1 つまでで、主柱の後ろに置いた
- [ ] 日本語の意図語がすべて英語化され、対応が判断メモに残っている (§3)
- [ ] §4 の矛盾類型で総当たりした (空間 / 温度 / エネルギー / 密度 / 年代 / ジャンル多重)
- [ ] 否定形・歌詞内容・実在アーティスト名が Style に混入していない (否定は Exclude へ)
- [ ] Exclude は少数精鋭で、Style と同語の矛盾がない
- [ ] タグ数と Style Influence の組み合わせが定石 (絞る × 高追従) に沿っている
- [ ] 文字数上限・タグ語彙・スライダー数値を suno-spec (実効版) で確認した
