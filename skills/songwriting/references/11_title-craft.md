# 11 タイトル設計 — 曲名の付け方

曲名は「聴く前に唯一見える言葉」であり、検索・共有・第一印象のすべてを背負う。この資料はタイトルの型・表記・造語・ダサさ回避・命名工程をまとめる。サビ頭/末への配置メカニクスとフック行からの切り出し (そのまま型/限定型/反転型) は 02 §2 が主管で、ここは「どんな型を、どの位置から、どう採り/採らず、どう審査するか」の総合資料。02 が配置、11 がタイトルの型と審査という分担にする。

## 0. この資料の約束

- 本文中の例タイトルはすべて自作、または一般類型としての記述であり、特定の実在曲を指さない。
- 実在曲のタイトルは §8 研究出典で「類型の実例」としてのみ言及する (タイトルは事実情報。歌詞原文は一切転載しない)。
- トレンドは古びる。命名の最新動向は suno-spec ではなくこの資料の範囲だが、鮮度は `:trend` 系リサーチで補完してから使う。

## 1. 大前提 — 「歌詞から採る=ダサい」ではない

調査の結論を最初に置く。歌詞 (特にサビ) からの抽出はプロの主流であり、洋楽ではフック=タイトルがデフォルト推奨ですらある。だから「歌詞の単語を使う」こと自体はダサさの原因ではない。

ダサくなるのは取り方の精度が低いときで、正体は次の 4 つ。

1. 機械的な切り出し (連呼語をそのまま採る、サビ頭を反射的に採る)
2. 説明過多 (テーマを直言してしまう)
3. 使い回しクリシェ (翼・奇跡・絆のような手垢のついた語の安易な結合)
4. 内容との不一致 (タイトル詐欺)

したがって本資料の方針は「歌詞流用の禁止」ではない。教えるのは、(a) どこをどう採るか (§2)、(b) 採らない選択肢=歌詞に出ない総括型を常に手札に持つこと (§2 の④)、(c) 型・表記・造語・審査という道具立て (§3〜§7) である。

発案者の直感との照合 (忖度せず正直に): 「歌詞の単語をそのまま使うパターンが多すぎる」という指摘は**部分的に反証された** — 問題は流用そのものではなく取り方にある。一方で「よほどキャッチーで語感の良いフレーズが歌詞にあればそのまま採る」「造語は 1 オプションで乱用しない」は調査でも支持された。

## 2. タイトルと歌詞の関係 (抽出位置)

タイトルをどこから採るかには 4 つの位置がある。近年の傾向まで込みで手札にする。

| 位置 | 何を採るか | 近年傾向 | 実務メモ |
|---|---|---|---|
| ① サビのリフレイン | 連呼される語をそのまま | 減少 | 最も付けやすいが機械的になりやすい。②③と比べてから決める |
| ② サビ終わりのキメ | サビ末の決め言葉 | 最多・推奨 | 現在の第一候補。サビの「残す 1 行」(02 §2) から採る |
| ③ 歌詞内キーワード | サビ以外の印象語 | 中間 | Verse/Bridge で一度だけ光る具体語を拾う |
| ④ 総括型 (歌詞に出ない) | 歌詞に出ない言葉で本質を言い当てる | 増加中 | 常に選択肢に入れる。説明ではなく「本質の一語」 |

- キャッチーで語感の良いフレーズが歌詞にあれば、そのまま採るのは強く有効である (検索での発見可能性が最大化する)。無理に捻らない。
- ④ の総括型 (歌詞に出ないタイトル) について、02 §2 はタイトル=フック一致を原則とし歌詞に出ない設計を「あえての判断」とするが、本資料はそれを常設の第 4 の選択肢として扱う。矛盾ではなく分担で、02 が配置の原則、11 が「採らない選択肢」の常備を担う。①②が凡庸なときは、④で本質を突けないか必ず一度試す。
- 配置 (サビ頭か末か) と反復回数は 02 §2 / §3、タイトルの完全一致検査は 08 §2-C。ここでは重複させず接続する。

## 3. 類型カタログ

12 の型。排他ではなく組み合わせ可能。例はすべて自作。

| 類型 | 効く理由 (1 行) | 使いどころ | 再現手順 (簡潔) |
|---|---|---|---|
| 一語・体言止め | 記憶・検索・言及が最速。ストリーミング期に構造的有利 | ジャンル問わず第一候補 | 感情/情景の名詞を 20 個出す → 口に出して締まる語を残す → 被り検索 |
| 漢字圧縮 (一〜四字) | 少ない字数に多義を詰め、硬質・文学的に見える | 激情・和・アニメ世界観 | 和語/古語/仏教語を探索 → 無ければ二字を新結合 → 読みが立つか確認 |
| ひらがな開き | 優しさ・素朴・儚さを表記で運ぶ | 内省・喪失・柔らかい曲 | 候補を漢字/カナ/かなで並べ書き → 感情温度に合う表記を選ぶ |
| 英語・カタカナ一語 | 検索性とジャンル感の即時シグナル | 洋楽感・K-POP 感・グローバル狙い | テーマを英訳 → 一語名詞化 → カナ音写も試し被りで選ぶ |
| 口語フレーズ | 当事者性が出てミーム化しやすい | SNS 拡散前提・キャラ立て | 主人公が実際に口に出す台詞を書く → 5〜10 字に刈る → 語尾で温度調整 |
| 文章型 (名詞+助詞+動詞) | 認識しやすさと独自性を両立し、物語が起動する | バラード・物語性の曲 | 核の名詞+動詞を選ぶ → 助詞を差し替えて語感比較 → 被り検索 |
| 長文型 | 一覧で物理的に目立ち、題だけで物語が完結する | 差別化の飛び道具 (乱発しない) | サビ核文/心内語を一文に → UI で切れても前半で意味が立つ語順に |
| 造語・当て字 | 検索独占。定着すれば強烈に残る (§5 が主管) | 唯一性が要るとき限定 | §5 の手順。審査を通ったときだけ採用 |
| 記号・スタイリング | 画面上で音量・テンションを伝える唯一の手段 (§4 が主管) | ダイナミクスを表記で示したいとき | §4。大小文字/記号ありなしを並べて選ぶ |
| 二重読み | 聴く前と後で意味が反転しリピートの発見を生む | 反転構造・伏線のある曲 | 候補にポジ読み/ネガ読みを書き出し両立語だけ残す |
| 異質結合 | 「どういう意味?」の謎がクリックを誘発する | フックの弱いテーマの底上げ | テーマ語を固定 → 遠い名詞を総当たり → 意味が発生する組だけ残す |
| 象徴物 (斜めに言う) | 説明を避け聴後の「そういうことか」を作る | テーマ直言を避けたいとき | 直言候補を全部捨てる → 歌詞世界に置ける具体物を列挙 → 遠いのに繋がる物を選ぶ (花言葉の逆引きも可) |

自作例: 一語「夜光」/ 漢字圧縮「灯芯」/ ひらがな「ほどける」/ カタカナ「アンダートウ」/ 口語「もう帰ろっか」/ 文章「星をこぼす」/ 長文「終電を見送って僕らはまだ何者でもない」/ 異質結合「制服とロケット」/ 象徴物 (別れを霜柱に託す)「霜柱」。

## 4. 表記の設計 (独立項目)

同じ音でも表記が変われば運ぶ感情が変わる。表記はメロディやモチーフと同格の、独立した設計項目として立てる。

| 表記 | 運ぶ感情・印象 |
|---|---|
| ひらがな | 優しさ・素朴・儚さ・脆さ。子供性・体温 |
| カタカナ | クール・無機質・記号性。日本語文脈では準造語化して被りが減る |
| 漢字 | 硬質・文学的・様式。字面で世界観を立てる |
| 英字 | スマート・グローバル・ジャンル感。小文字=内省、全大文字=激情の目安 |
| 記号 (! 。 、 …) | 音のない画面で音量・余韻・勢いを刻む |

- 決定稿は必ず 3〜4 表記で並べ書きし、曲の感情温度に合う 1 つを選ぶ。
- **アーティスト名と横並びで見る**。プレイリストやジャケットでは常に「アーティスト名 — 曲名」の対で表示される。名前の表記 (artist.yaml) と衝突・重複しないか、対で見て締まるかまで確認する。
- 日本語曲でも英字・カタカナ題は有効だが、意味の裏付けなしに「スマートに見せたいだけ」で変換すると空回りする (§6)。
- 表記いじりの最終形は入稿版の変換規則 (06 の R 群) とは別物である。作品版タイトルの表記が正で、入稿時の発音対策とは分けて考える。

## 5. 造語の扱い (1 オプション、乱用しない)

造語には強さと弱さの両面がある。既定で造語に寄せない。

グラデーション (どれも「ゼロからの音の羅列」より、意味の乗った半造語が定石):

1. 音写準造語 (外来語の音をカナで写す。表記で唯一性を得る)
2. 逆転・再結合 (既存の複合語を組み替える)
3. 架空固有名 (実在しない地名・人名)
4. 漢字新結合 (実在熟語がなければ二字を組む)

- 強み: 検索を独占できる。定着すれば強烈に記憶される。
- 弱み: 認知に時間がかかる。読めない語・複数読みの語 (アイ/エーアイ問題) は検索も口コミも割る。
- **結論**: 造語は 1 つの強力なオプションであって全体戦略ではない。曲ごとに既定で造語へ寄せず、§7 の審査を通ったときだけ採用する。造語だらけの作家性はこのプロダクションでは狙わない (発案者の要望と調査が一致した点)。
- 例外: 難読・造語が「引き」になるニッチ (ボカロの熟語美学など) は、ジャンルの読解リテラシーに依存する。ターゲット層を確認してから使う。

## 6. ダサさの禁止パターン表

| 禁止パターン | 症状 | 代わりにどうするか |
|---|---|---|
| テーマ直言・説明過多 | 「愛」「別れ」等をそのまま置く、状況を説明した題 | 象徴物型 (§3) で斜めに言う。20〜30% の謎を残す |
| 使い回しクリシェ | 翼・奇跡・絆・涙の手垢のついた結合 | 具体物・固有イメージに置換。異質結合 (§3) で更新する |
| 時代感のズレ | ネットスラング・流行語で経年劣化する | 語尾/スラングは風化前提。普遍語に核を置き、流行は薬味に留める |
| 裏付けのない表記いじり | 意味もなく安易にカタカナ化・英字化する | 表記が運ぶ感情 (§4) と曲の温度が一致するときだけ変える |
| 内容との不一致 (タイトル詐欺) | 題と歌詞世界がずれている | §7 の内容整合審査で落とす。歌詞完成後に再照合する |

留保: ダサさは強く主観的であり、「型として正しい」ことと「万人に刺さる」ことは別である。審査は候補を落とすためではなく、外す理由を言語化するために使う。

## 7. 命名ワークフロー

天啓ではなく工程で作る。

1. **多案出し**: 10〜20 案を目安に、複数の類型 (§3) をまたいで出す。全部を同じ型・同じ位置 (サビ頭) に寄せない。②サビ末・③印象語・④総括型を必ず混ぜる。
2. **客観審査**: 各案を次で絞る。
   - 見た目: アーティスト名と横並びにする (§4)
   - 響き: 声に出す。歌ものとして締まるか
   - 表記比較: 漢字/かな/カナ/英字/記号を並べる (§4)
   - 検索被り: 配信・動画で候補名を検索する。強い既存曲と被れば語形・表記を変える (完全一致検査は 08 §2-C)
   - 内容整合: 歌詞世界と一致するか (タイトル詐欺の排除)
3. **3 秒の第一印象**: 聴き手は数秒で判断する。前提知識のない目で見て「気になる」か。
4. **仮タイトル運用**: 仮題で書き始め、歌詞完成後に §2 の位置と §3 の型で再審査してよい (詞を書き切ってから最後に付ける後付け型も実務的)。
- 納品では、審査を通った**本命 1 + 対抗 1〜2** に絞って提示する (作詞家の【タイトル案】)。
- **造語・凝った型 (二重読み・異質結合・長文) は、この審査を通ったときだけ採用する**。通らなければ一語・体言止めや文章型の平明な案に戻す。
