# 08 オリジナリティ規範と自己検査 — 納品前の最終関門

この検査はオリジナリティ担保 2 層の**第 2 層 (制作エージェントの自己検査)**。第 1 層 = エージェント定義の絶対規則 (執筆中つねに有効)、第 2 層 = この自己検査 + マネージャー側の機械検証 (validate_song.py)。**外部 Web 照合は v0.5.0 で廃止**したので、自己検査が最終防波堤になる。役割は「不安を隠さず可視化し、**自分で書き直すか、確信が持てなければ P に握らせる**」こと。**検査を通すことではなく、不安を可視化することが目的** — 迷ったら申告に書く。

## 1. 絶対規則 (第 1 層の再掲。違反は検査以前の問題)

- 実在曲を参考にしてよいが、**歌詞原文・特徴的なメロディラインの直接流用は一切禁止** (引用・翻案・穴埋め改変・「あの曲っぽい言い回し」を思い出しながら書くことを含む)。ブリーフのモチーフから組み立てる
- 実在のアーティスト名・曲名・バンド名を歌詞・タイトルに書かない (言及は自己申告の出典欄のみ)
- 有名曲のサビ 1 行レベルの一致・タイトルの完全一致を避ける (一般語の部分一致は許容)
- 影響を受けた「雰囲気」の言語化は OK。それを P に隠さない (自己申告文化)

## 2. 検査手順 (3 観点)

### A. フレーズ検査 (行単位)

1. **検査対象行を抽出する**: サビ頭行 / サビ末行 / タイトルを含む行 / 決め台詞 (Bridge の告白行など) / 自分で「強い」と感じる行 — 計 4〜8 行
2. 各行について「**この行はどこから来たか**」を 1 行で答える。ブリーフのモチーフからどの展開 (04 §2) を経てこうなった、と説明できれば健全
3. **記憶由来を疑う兆候**が 1 つでもあれば、その行は要対処:
   - 書いた瞬間に、自分が設計した譜割りとは別のメロディ付きで浮かんだ
   - 特定の既存曲・アーティストの顔が浮かぶ
   - 「どこかで聞いた気がする」のに出所を説明できない
   - 慣用句を超えた長さ (7 モーラ以上 / 英語 4 語以上) の言い回しが一発で出てきて、修正の跡がない
4. 対処: §4 の書き直しプロトコルを実行するか、確信が持てなければ**自己申告の「不安のある行」に載せる** (自分で書き直すか、P に握らせる判断材料にする。Web 照合は廃止されたので、原則はこの場で書き直す)

補足: 頻出の一般表現 (「君に会いたい」「夜を越えて」級) は一致しても権利上の問題にならない。ただし一般表現だけで組まれた行は摩耗 (04 §6) として品質側で直す。

### B. 構造検査 (曲単位)

1. 構成表・感情曲線・モチーフ展開 (提示 / 反転 / 回収の位置) を並べて眺める
2. 「この並びは特定の 1 曲をそのままなぞっていないか」を自問する。**Verse–Chorus 型や V 字曲線のような定型・類型は共有財産なので問題ない**。問題になるのは特定曲固有の意匠の一致 (例: 特徴的な語りの入り方 + 同じ位置の転調 + 同じモチーフの回収、が全部揃う)
3. 特定曲が想起されたら: 構成要素を 1 つ以上ずらす (Bridge の内容を変える、回収位置を変える)。ずらさずに残す判断をしたなら「雰囲気の影響」として自己申告に書く

### C. タイトル検査

1. タイトルが**有名曲と完全一致していないか**を記憶ベースで確認する
2. 一般語 1〜2 語のタイトル (「サクラ」「ハナビ」級) は同名曲が多数存在しうる。完全一致が直ちに問題になるわけではないが、検索で埋没する実害があるため、限定を足した差別化案を併記する (「サクラ」→「終電のサクラ」)
3. 確信が持てない場合はタイトルも「不安のある行」に含めて申告し、差別化案を併記する (Web 照合は無いので、完全一致は自分で避ける)

### 補助検査: 自分の過去曲との重複 (アーティスト文脈のみ)

継続アーティストでは、他人の曲だけでなく**自作の使い回し**も見る (同じ決め行が 2 曲にあると、ディスコグラフィー全体の資産価値が下がる)。

- ブリーフに直近曲の要約や過去曲のフック行が含まれていれば、今回のサビ頭・サビ末との一致を確認する
- 情報が渡されていなければ検査不能なので、自己申告に「過去曲との重複は未検査 (情報なし)」と 1 行残す (渡されていないファイルを推測で探さない)

## 3. オリジナリティ自己申告 (納品時に必ず添える)

納品フォーマットの末尾に、次の 3 項目を埋めて添える。

```text
【オリジナリティ自己申告】
- 中心モチーフの出所: (ブリーフのどの要素から、どの展開パターンでこの表現に至ったか。1〜3 行)
- 意図的に避けた既存曲: (執筆中に接近を自覚して外した曲・言い回し。なければ「なし」)
- 不安のある行: (検査 A / C で確信が持てなかった行の全文を列挙。原則はこの場で書き直し、書き直せない/確信が持てないものだけ残して P に握らせる。なければ「なし」)
```

記入基準:

- 「なし」と書けるのは**検査を実施した上で該当がなかった**ときだけ (検査を省略しての「なし」は禁止)
- 「不安のある行」の閾値は低くする。**事故コストは信頼問題として高い**一方、外部照合は無いので、迷う行は自分で書き直すのが原則 — 書き直せないものだけ申告に残す
- 「意図的に避けた既存曲」には実在の曲名を挙げてよい (歌詞・タイトルへの使用ではなく、P への申告のため)。この欄は品質情報でもある (マネージャーが P に「この曲とは意図的に離してあります」と説明できる)

記入例 (粒度の目安。自作の想定曲):

```text
【オリジナリティ自己申告】
- 中心モチーフの出所: ブリーフの「夜の街 / 距離」から連想マップで「街の灯り」を採用。展開は反転型 (灯り = 存在の印 → 消えた窓 = 不在の印)
- 意図的に避けた既存曲: なし (執筆中に特定曲の想起はなかった)
- 不安のある行: 「まちのあかり ほどけて」— 灯りと「ほどける」の組み合わせをどこかで聞いた気がするが、出所を特定できない。§4 の書き直しプロトコルで別の具体物に振り直した
```

## 4. 書き直しプロトコル (検査 NG 時)

対象行が既存曲に近いと感じたときの手順。

1. その行が担っていた**機能**を構成表で確認する (フック / 傾斜 / 回収など)
2. 表現をいったん捨て、**ブリーフのコアモチーフに戻って別の具体物・別の動作で言い直す** (04 §6 の感情語翻訳と同じ要領)。元の行の語を 1〜2 語だけ差し替える「ずらし」は翻案になりうるため不可
3. モーラ数 (英語はストレス数) と韻計画は元の行に合わせて維持する (譜割りと構成を壊さない)
4. 書き直した行を検査 A にかけ直す
5. 修正が作品版に入ったら、入稿版を再変換する (06 の三原則)

## 5. 検査の出口条件

- [ ] A: 検査対象行 (4〜8 行) のすべてに「出所」の説明が付いた
- [ ] A: 記憶由来の兆候がある行は、書き直したか申告に載せた
- [ ] B: 特定曲の丸なぞりがない (定型の使用は問題ない)
- [ ] C: タイトルの完全一致確認を行い、必要なら差別化案を併記した
- [ ] アーティスト文脈では過去曲との重複を確認した (情報がなければ「未検査」と申告した)
- [ ] 自己申告 3 項目が「検査を実施した上で」埋まっている
