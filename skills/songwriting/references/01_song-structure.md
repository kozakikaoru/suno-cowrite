# 01 曲構成理論 — 器を先に決める

歌詞の失敗の多くは行の質ではなく、器 (構成) の選び間違いで起きる。この資料は、構成の型を選び、Suno のメタタグ列に写像し、歌詞量を決めるまでの手順。ワークフロー段階 1 (設計) で使う。

## 1. 基本 3 型と使い分け

### Verse–Chorus 型 (現代ポップスの標準)

- 形: Intro → Verse 1 → (Pre-Chorus) → Chorus → Verse 2 → (Pre-Chorus) → Chorus → Bridge → Chorus → Outro
- 性質: サビ (Chorus) が曲の顔。反復で刷り込む
- 選ぶ条件: フック勝負の曲、シングル志向、ショート動画展開の想定。**迷ったらこれ**

### AABA 型 (スタンダード/バラード系)

- 形: A → A → B → A。サビという概念がなく、A 自体が主役
- 性質: 物語が静かに進む。B で一度だけ視点が変わる
- 選ぶ条件: 語り中心のバラード、ジャズ・歌謡曲的な質感、サビの爆発が世界観に合わない曲
- Suno 上の注意: `[Chorus]` を使わず、A を `[Verse 1]` `[Verse 2]`… で並べ、B は `[Bridge]`。A 間のメロディの統一感は、行数とモーラ数を A 同士で揃えることで担保する (番号付きタグは別展開に振れやすいため。§3)

### EDM 型 (Build–Drop)

- 形: Intro → Verse → Build → Drop → Verse → Build → Drop → (Bridge) → Drop
- 性質: 感情のピークは歌詞ではなく Drop (音) が担う。歌詞は Build までの助走
- 選ぶ条件: ダンス/エレクトロ系。Drop はボーカルなし、または短いチャントのみ
- Suno 上の注意: `[Build]` `[Drop]` タグを使う。Drop 区間には歌詞を書かず、`[Drop]` 単独か、チャント 1 行 (02 §5) だけを置く

### 選択フロー

1. ブリーフに構成案があればそれに従う (作詞家が勝手に変えない。変えたい理由があれば納品時に提案として書く)
2. なければ: ダンス主体 → EDM 型 / サビで開く曲 → Verse–Chorus 型 / 静かな物語 → AABA 型
3. 型は混ぜない。「AABA だがサビもある」のような折衷は、Suno がどちらの構成にも寄れず崩れやすい。迷ったら Verse–Chorus 型に寄せ切る

## 2. J-POP 定型と英米ポップの対応表

| J-POP 用語 | 英米ポップ用語 | 役割 |
|---|---|---|
| イントロ | Intro | 世界に入る導入 |
| A メロ | Verse | 状況・風景の提示 |
| B メロ | Pre-Chorus | サビへの傾斜。緊張を上げる |
| サビ | Chorus | 感情と主題のピーク。タイトル/フックの置き場 |
| 間奏 | Interlude / Instrumental Break | 呼吸。場面転換 |
| C メロ | Bridge | 一度だけの視点転換・飛躍 |
| 落ちサビ | Stripped chorus (Final Chorus の前段) | 静めてから最後を最大化する助走 |
| 大サビ / ラスサビ | Final Chorus | 最後の最大サビ |
| アウトロ | Outro | 余韻 |

J-POP 定型のフル形: A → B → サビ → A' → B' → サビ → C (または落ちサビ) → 大サビ → アウトロ。

## 3. Suno メタタグへの写像表

構成表のセクション列を、そのままタグ列に変換する。タグは英語で、**suno-spec の語彙一覧にあるものだけ**使う。

| 構成表の書き方 | Suno タグ列 |
|---|---|
| イントロ (歌なし) | `[Intro]` (歌詞行を置かない) |
| A メロ 1 / 2 | `[Verse 1]` / `[Verse 2]` |
| B メロ | `[Pre-Chorus]` |
| サビ | `[Chorus]` |
| サビ後の畳みかけ | `[Post-Chorus]` |
| 間奏 | `[Instrumental Break]` (楽器を立てるなら `[Guitar Solo]` 等) |
| C メロ | `[Bridge]` |
| 落ちサビ | `[Bridge: stripped down, quiet]` + 直後に `[Chorus]`、または `[Chorus: stripped down, piano only, soft vocals]` |
| 大サビ | `[Chorus]` (持ち上げるなら直前に `[Key Change]`、または `[Chorus: full band, soaring vocals]`) |
| ラップ区間 | `[Rap]` (03 §5) |
| 語り | `[Spoken Word]` |
| アウトロ | `[Outro]` (フェードで閉じるなら `[Fade Out]` 併用、確実に終止させるなら `[End]`) |

### タグ運用の原則 (Suno 特性)

- **タグを置かないと歌詞全体が一塊として処理され、構成が崩れやすい**。全セクションに必ずタグを置く。
- **番号付きタグは「別展開」の指示**: `[Verse 1]` と `[Verse 2]` は異なるメロディ展開に振れやすい。1 番と 2 番の歌詞内容も変える (05 §3) のが噛み合う。
- **同一歌詞の `[Chorus]` 反復は「同じメロディ」の指示**: サビのメロディを固定したいなら、サビ歌詞は全サビで一字一句同じにするのが最も確実。サビごとに歌詞を変えるのはメロディも変わるリスクを取る行為 (妥協点は 05 §5 の 1 語差し替え)。
- **パラメータ付きタグ `[Chorus: stripped down, piano only]` はセクション単位の演出指示**。Style 欄を変えずに音を動かせる、実用上最も強力な制御手段。感情曲線との連動は 05 §6。
- タグが稀に歌われる事故は確率的でゼロにできない。事故が出たら該当タグを標準語彙内の別表現に変えて再生成する (06 第 3 部)。

### タグ列の実例 (1 曲分のスケルトン)

§5 の構成表をタグ列に落とすと次の形になる。設計段階でこのスケルトンまで作り、歌詞は空欄のまま執筆段階に渡す。

```text
[Intro]

[Verse 1]
(6 行)

[Pre-Chorus]
(4 行)

[Chorus]
(6 行)

[Verse 2]
(6 行)

[Pre-Chorus]
(4 行)

[Chorus]
(1 回目と同一歌詞)

[Bridge: stripped down, vulnerable vocals]
(4 行)

[Chorus: full band, soaring vocals]
(末行のみ 1 語差し替え)

[Outro]

[End]
```

## 4. テンポ別・曲長別の歌詞量目安 (日本語)

**詰め込みは早口化を、不足は早終了を招く** (Suno は歌詞量と行密度から曲の尺を推定する)。器に合った量を設計段階で決め、執筆で守る。

| テンポ帯 | 1 行の目安 (モーラ) | セクション行数の目安 | 3〜4 分想定の本文総量 (タグ除く) |
|---|---|---|---|
| スロー (〜80 BPM 感) | 6〜10 | Verse 4〜6 行 / Chorus 4〜6 行 | 250〜450 字 |
| ミッド (80〜115 BPM 感) | 8〜12 | Verse 6〜8 行 / Chorus 6〜8 行 | 400〜700 字 |
| アップ (115 BPM 感〜) | 10〜16 | Verse 8 行前後 / Chorus 8 行前後 | 550〜900 字 |

- Pre-Chorus は 2〜4 行、Bridge は 2〜6 行が目安 (どのテンポ帯でも)。
- 上限値は suno-spec が正 (現行スナップショット: 上限 5,000 字・実用 3,000 字以下推奨)。上表の範囲なら大きく余裕がある。3,000 字に近づくのは長尺曲か詰め込みのどちらかで、たいてい後者。
- 曲長の調整:
  - 短くしたい (2 分台) → 削る順は「2 番の Pre-Chorus → Post-Chorus → 間奏 → 2 番 Verse の後半」。**サビは削らない**
  - 長くしたい (5 分前後) → 間奏・Post-Chorus・Bridge を足す。Verse を 3 つにするより反復と間奏で伸ばす方が破綻しない
- 配分のよくある失敗:
  - サビだけ行数が多く Verse が薄い → サビ前で息切れした印象になる。行数配分は構成表で先に固定する
  - 全セクションが同じ行数 → 呼吸がなく単調になる。Pre-Chorus と Bridge は短くして緩急を作る
- 英語はモーラではなくストレス数で数える (07 §4)。行数の目安は同じ。

## 5. 構成表テンプレート (設計の成果物)

設計段階で次の表を作ってから執筆に入る。例はミッドテンポ・V 字型の想定。

| # | セクション | タグ | 行数 | 1 行モーラ | 感情強度 (05) | 役割・モチーフ (04) |
|---|---|---|---|---|---|---|
| 1 | イントロ | `[Intro]` | 0 | — | 2 | 楽器のみ |
| 2 | A メロ 1 | `[Verse 1]` | 6 | 8〜10 | 3 | 夜の街の風景提示 |
| 3 | B メロ | `[Pre-Chorus]` | 4 | 8 | 5 | 疑問形で傾斜 |
| 4 | サビ | `[Chorus]` | 6 | 10〜12 | 8 | タイトル回収・フック |
| 5 | A メロ 2 | `[Verse 2]` | 6 | 8〜10 | 4 | 時間経過後の風景 |
| 6 | B メロ 2 | `[Pre-Chorus]` | 4 | 8 | 6 | 決意の芽 |
| 7 | サビ | `[Chorus]` | 6 | 10〜12 | 8 | 同一歌詞で反復 |
| 8 | C メロ | `[Bridge: stripped down, vulnerable vocals]` | 4 | 6〜8 | 3 | 反転・告白 |
| 9 | 大サビ | `[Chorus]` | 6 | 10〜12 | 10 | 末行のみ 1 語差し替え |
| 10 | アウトロ | `[Outro]` | 0〜2 | — | 4 | 余韻 |

表を書き終えたら執筆前に検査する:

- [ ] タグがすべて suno-spec の語彙一覧にある
- [ ] 行数 × モーラの概算が §4 のテンポ帯総量に収まる
- [ ] 感情強度が平坦になっていない (05 §1 のルール)
- [ ] 1 回目のサビが曲頭から 60〜90 秒相当 (イントロ + Verse + Pre-Chorus が長すぎない) に来る (02 §6)
