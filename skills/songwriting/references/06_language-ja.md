# 06 日本語作詞 — 歌える日本語と Suno 入稿変換 (完全版)

この資料は 3 部構成。**第 1 部 = 日本語で「歌える」歌詞を書く技術** (作品版の質)、**第 2 部 = 作品版 → 入稿版の機械的変換規則** (Suno の発音事故対策)、**第 3 部 = 生成後のガチャ運用**。表現の質は作品版で追い、発音対策は入稿版で吸収する — この 2 つを混ぜない。

- **作品版**: 漢字かな交じり。P への提示・YouTube 概要欄・字幕に使う
- **入稿版**: ひらがな化・助詞ローマ字化・括弧排除済み。Suno の Lyrics 欄に貼る唯一の版

---

## 第 1 部: 日本語で「歌える」歌詞を書く

### 1. モーラ感覚と字余り

- 日本語の歌は音符 1 つにモーラ 1 つが基本 (数え方は 03 §4)。
- 字余り (割付 ±1〜2 モーラ) はメロディ側が細かい音符で吸収するので許容される。ただし Suno では**行が長いほど詰め込まれる**ため、迷ったら字余り方向ではなく字足らず方向に倒す方が事故が少ない。
- 助詞を 1 つ削ると 1 モーラ稼げる (§3)。音数合わせのための無意味な「ああ」「そう」の挿入は最後の手段。

### 2. 母音とサビ適性 (伸ばしどころの設計)

日本語は開音節言語で、**行末・フレーズ末の母音が伸ばされやすい**。サビの決め所は、伸ばして映える母音で終わる語を選ぶ。

| 行末の音 | 伸ばしたときの質感 | サビ末適性 |
|---|---|---|
| a 段 (〜だ、〜な、〜か) | 開放・明るい | ◎ |
| o 段 (〜よ、〜も、〜ろう) | 温かい・包む | ◎ |
| e 段 (〜ね、〜て) | 呼びかけ・切なさ | ○ |
| i 段 (〜い、〜み) | 締まる・鋭い | ○ (切実系の曲) |
| u 段 (〜う、〜く) | こもる | △ |
| ん / っ | 伸ばせない | × (サビ末では避ける) |

- 悪い例 → 良い例 (サビ末。自作): 「きみを待ってたんだけど」(末尾が「ど」で濁って重い) → 「きみを待ってた 今も」(o 段で開放して伸ばせる)。
- Verse の行末は多様でよい。**サビの最後の 1 音だけは必ず設計する**。

### 3. 口語/文語と助詞省略

- 基準はアーティストの口調 (character.md)。口語基調のアーティストに文語 (「〜せし」「〜ゆえに」) を混ぜると浮く。文語を使うのは世界観設定に根拠があるときだけ。
- **助詞省略は歌詞の標準技法**: 「街の灯りが滲む」→「街の灯り 滲む」。省略で生じた間 (字空け) がそのままブレスになる。
- 省略しすぎの兆候: 1 行に裸の名詞が 3 つ以上並ぶ (「夜 街 灯り 君」)。単語の羅列は AI 作詞にありがちな既視感パターンでもあるため避ける。
- 残すべき助詞: 意味の対立を担う助詞は削らない (「君**が**いない」と「君**は**いない」の差が立つ場面)。

### 4. 語尾のバリエーション (単調の回避)

- 同じ語尾が 3 行連続したら黄信号 (韻として設計した場合を除く。03 §2)。
- 語尾の持ち駒: 「〜た」(完了) / 「〜る」(現在) / 「〜て」(継続・余韻) / 「〜ない」(否定) / 体言止め / 依頼・命令 (「〜して」) / 疑問 (「〜の?」) / 呼びかけ (「〜よ」「〜ね」)。
- 悪い例 → 良い例 (自作):
  - 悪い: 「歩いた / 探した / 泣いた / 笑った」(「た」4 連続)
  - 良い: 「歩いた / 探してる / 泣かないよ / 笑ってみせる」(完了 → 現在 → 否定 + 呼びかけ → 意志)
- Pre-Chorus の末尾は「言い切らない形」(〜て、〜まま、体言止め) にするとサビへ傾斜する。

### 5. 作品版での英単語・カタカナ語

- 原則: **1 曲に英語フレーズの系統は 1 つまで** (サビの決めフレーズか、合いの手のどちらか)。あちこちに散らすと安っぽくなる。
- 「日本語で言える語をわざわざ英語にする」のは摩耗表現の代表格。使うなら日本語で置換できない理由を持つ。
- カタカナ語 (ネオン、ガラス、メロディ) は日本語の語彙として自由に使ってよい。入稿時の扱いは第 2 部 R6。
- チャント (oh oh / la la) は小文字ローマ字で書く (02 §5)。これは英語フレーズの系統には数えない。

## 第 2 部: Suno 入稿変換規則 (作品版 → 入稿版)

### 変換の三原則

1. **一方向**: 変換は常に「作品版 → 入稿版」。入稿版を直接編集しない (直したくなったら作品版を直して再変換する)
2. **機械的**: 変換中に表現判断をしない。下の規則表 R1〜R9 を番号順に適用するだけ
3. **行構造の保存**: 変換で行の数・順序・分割を変えない (行構造 = 譜割り。03 §4)

### 変換の実行手順 (毎回この 5 ステップ)

1. **読み確定メモ**: 作品版の多読み語・数字・英単語を列挙し、R1 / R5 / R6 の決定を先にメモする
2. **機械変換**: R1〜R9 を番号順に全行へ適用する (1 行ずつ。表現はいじらない)
3. **チェックリスト**: 下の変換チェックリストを全項目確認する
4. **逆読みテスト**: 入稿版だけを音読し、作品版と同じ歌詞に聞こえるか確かめる
5. **凍結**: 以後の修正は作品版に入れて 1 からやり直す (入稿版への直接パッチは禁止)

### 変換規則表 (R1〜R9、適用順)

| # | 規則 | 内容 |
|---|---|---|
| **R1** | 読みの確定 | 多読み語 (下の判断基準表) の読みを、作品版の意図どおりに確定する。判断基準: (1) 設計のモーラ割付に合う方 (「あす」= 2 モーラ / 「あした」= 3 モーラ)、(2) アーティストの口語/文語トーン (「ゆく」= 文語寄り / 「いく」= 口語)、(3) 迷ったら声に出して自然な方 |
| **R2** | 漢字の全ひらがな化 | R1 で確定した読みで、**漢字をすべてひらがなに開く** (固有名詞も含む)。カタカナ語はカタカナのまま残す (読みが一意のため)。同音異義の紛れは無視してよい (Suno は音だけを歌う) |
| **R3** | 助詞のローマ字化 | 係助詞「は」→ `wa`、格助詞「へ」→ `e`、格助詞「を」→ `wo`。**前後に半角スペース**を入れる (「きみ wo さがす」)。語の一部の は/へ/を (「はな」「へや」) は対象外。細則は下記 |
| **R4** | 括弧・ルビの全排除 | `()` `（）` `「」` `『』` `《》` と、タグ以外の `[]` をすべて除去する。**括弧はコーラス/リフレイン指示として誤解釈されうる** (Suno 仕様)。セリフの「」は外して地の文にする。ルビ・当て読み (「本気」と書いて「マジ」と読ませる類) は、読ませたい音の方だけを書く |
| **R5** | 数字の読み下し | 算用数字・漢数字を読みのひらがなに開く: 「2 人」→「ふたり」、「17 歳」→「じゅうななさい」、「100 万回」→「ひゃくまんかい」。英語で読ませたい数字だけ英語綴り (「seventeen」)。年号などの長い数字は読みを設計時に決めておく (「2049」→「にせんよんじゅうきゅう」) |
| **R6** | 英単語の扱い | 英語発音させたい語は**英語綴りのまま**残し、前後に半角スペースを置く (「ひかりの melody」)。日本語発音させたい外来語は**カタカナ**にする (「メロディ」)。頭字語 (DNA、SOS) は発音が不安定なのでカタカナ読みに開く (「ディーエヌエー」「エスオーエス」) |
| **R7** | 記号の整理 | 「…」「―」「〜」は除去する (音を伸ばしたい箇所は長音「ー」で書く: 「ずっとー」)。「!」「?」は残す場合は半角にする (歌唱への影響は軽微)。読点「、」句点「。」は除去し、間は字空けか行分割で表す |
| **R8** | スペースの整理 | 作品版の字空けは維持する (ブレス位置のヒント)。R3 / R6 で入れた半角スペースの連続は 1 つにまとめる。行頭・行末の余分なスペースは削る |
| **R9** | タグの保全 | 構成タグ・パラメータ付きタグ (`[Chorus: stripped down]` 等) は英語のまま一切変更しない。タグが suno-spec の語彙一覧にあることを最終確認する |

### R1 用: 多読み語の判断基準表 (代表例)

| 表記 | 読み候補 (モーラ数) | 選び方の目安 |
|---|---|---|
| 明日 | あす (2) / あした (3) | モーラ割付を優先。「あす」はやや文語、「あした」は口語 |
| 行く | いく (2) / ゆく (2) | 歌では「ゆく」が滑らかで伸ばしやすい。口語感なら「いく」 |
| 何 | なに (2) / なん (2) | 続く語で決まる (「なにを」「なんで」)。誤読されやすいので必ず開く |
| 二人 | ふたり (3) / ににん (3) | 歌詞ではほぼ「ふたり」。誤読防止のため必ず開く |
| 一人 | ひとり (3) / いちにん (4) | 同上。「ひとり」 |
| 今日 | きょう (2) / こんにち (4) | ほぼ「きょう」。開かないと誤読リスクが残る |
| 昨日 | きのう (3) / さくじつ (4) | ほぼ「きのう」 |
| 風 | かぜ (2) / ふう (2) | 単独なら「かぜ」。「あんな風に」は「あんなふうに」と開いて書き分ける |
| 空 | そら (2) / くう (2) / から (2) | 「そら」以外の意図なら特に注意して開く |
| 方 | ほう (2) / かた (2) | 文脈で割れる代表格。必ず開く |
| 辛い | つらい (3) / からい (3) | 文脈で判別不能な代表格。必ず開く |
| 間 | ま (1) / あいだ (3) | モーラ数が大きく変わるため割付と必ず照合する |
| 今 | いま (2) / こん | 単独なら「いま」。熟語は熟語ごと開く (「今夜」→「こんや」) |
| 家 | いえ (2) / うち (2) | 建物なら「いえ」、生活の場なら「うち」。口調設定とも照合する |
| 月 | つき (2) / がつ (2) / げつ (2) | 空の月は「つき」。日付は R5 と併せて開く (「三月」→「さんがつ」) |
| 後 | あと (2) / のち (2) / うしろ (3) | 三方向に割れる代表格。必ず開く |

原則: 表にない語でも、**読みが 2 つ以上ありうる漢字はすべて開けば安全**。R2 で全ひらがな化するため、この表は「どの読みで開くか」を決めるためだけに使う。

### R3 用: 助詞ローマ字化の細則

- 対象は**助詞として機能している は/へ/を だけ**:
  - 「わたし**は**」→「わたし wa」 (対象)
  - 「**は**な (花)」→「はな」のまま (語の一部なので対象外)
  - 「こんにち**は**」→ 挨拶語は分割せず、音のとおり「こんにちわ」に書き換える
- 複合形は「は」の部分だけ変換する: 「〜では」→「で wa」、「〜とは」→「と wa」、「〜へは」→「e wa」
- 「へ」は方向の格助詞のみ対象 (「きみのもと e」)。「へや (部屋)」等は対象外
- 「を」は目的格をすべて `wo` にする (「よる wo こえて」)
- スペース規則: ローマ字助詞の前後に半角スペース 1 つ。行末に来る場合は後ろのスペース不要

### 変換の全行例 (ビフォーアフター。自作)

作品版:

```text
[Verse 1]
街の灯り ほどけて
君を探す 夜の底
「まだ間に合う」と 呟いた
2 人分の 明日へ
```

入稿版:

```text
[Verse 1]
まちのあかり ほどけて
きみ wo さがす よるのそこ
まだまにあうと つぶやいた
ふたりぶんの あした e
```

適用された規則: R1 (明日 → 「あした」を選択 ※3 モーラ割付のため / 間に合う → 「まにあう」)、R2 (全ひらがな化)、R3 (を → wo、へ → e)、R4 (「」の除去)、R5 (2 人 → ふたり)。行数は 4 → 4 で不変 (三原則 3)。

もう 1 例 (数字・英単語・係助詞を含むサビ。自作):

作品版:

```text
[Chorus]
午前 2 時の melody
君へ 鳴らす 100 回目
今夜は 眠らないまま
「おやすみ」は 言わない
```

入稿版:

```text
[Chorus]
ごぜんにじの melody
きみ e ならす ひゃっかいめ
こんや wa ねむらないまま
おやすみ wa いわない
```

適用された規則: R3 (へ → e、係助詞 は → wa を 2 箇所)、R4 (「」の除去)、R5 (2 時 → にじ、100 回目 → ひゃっかいめ)、R6 (melody は英語発音のまま維持)。

### 変換チェックリスト (全項目必須)

- [ ] 漢字が 1 文字も残っていない (固有名詞を含む)
- [ ] 助詞の は/へ/を がすべて `wa` / `e` / `wo` になっている (語中の は/へ/を を誤変換していない)
- [ ] あらゆる括弧 `()` `（）` `「」` `『』` が残っていない (タグの `[]` を除く)
- [ ] 算用数字・漢数字が残っていない
- [ ] 「…」「〜」「―」が残っていない (伸ばしは「ー」)
- [ ] 英字の連続が「意図した英単語」だけになっている (頭字語はカタカナ化済み)
- [ ] 行数・行順が作品版と完全に一致している
- [ ] タグがすべて suno-spec の語彙一覧にある
- [ ] 多読み語の読みが設計のモーラ割付と一致している
- [ ] 逆読みテスト: 入稿版を音読すると作品版と同じ歌詞に聞こえる (表現を変えていない)
- [ ] 本文総字数が実用目安 (約 3,000 字) 以下 — 上限値の最新は suno-spec を確認

## 第 3 部: ガチャ運用の心得 (生成後の修正対応)

同一入力でも生成ごとの揺らぎが大きい。**1 回で完成させる前提を捨て、修正対象を 1 箇所ずつ潰す**。

### 症状別の対処表 (P からのフィードバック対応)

| 症状 (P の報告例) | 原因の見立て | 歌詞側の対処 |
|---|---|---|
| 「2 番 A メロが早口」 | 該当セクションの詰め込み | 該当セクションだけ総モーラを 2〜4 割減らす (行を削るか各行を短くする)。他のセクションは触らない |
| 「間延びする」「曲が早く終わる」 | 歌詞量の不足 | 該当セクションに行を足す (Verse 4 行 → 6 行など)。01 §4 の総量目安に近づける |
| 「読み方が変」 | 表記の解釈揺れ | 該当語だけ表記を変える: ひらがな → ローマ字 (「こころ」→「kokoro」)、字空けの追加、カタカナ化 |
| 「サビのメロディが毎回違う」 | サビ歌詞の不一致 | 全サビの歌詞を一字一句同一にする (大サビの差し替えは 1 箇所まで。05 §5) |
| 「構成が無視される」 | タグ不足・語彙外タグ | 全セクションのタグの有無と、suno-spec の語彙一覧への適合を確認する |
| 「タグが歌われた」 | 確率的な事故 | 該当タグを標準語彙内の別表現へ変える (再生成だけで直ることも多い) |
| 「歌詞の一部が飛ばされる」 | 行の詰め込み・記号の残存 | 該当行を分割し、変換チェックリストを再実行する |
| 「英単語が日本語なまりになる」 | 英語綴りの解釈揺れ | 前後の半角スペースを確認し、より一般的な綴りに変える。日本語読みでよいならカタカナへ (R6 の再適用) |
| 「助詞が『ハ』『ヘ』と歌われる」 | R3 の変換漏れ | 該当行の は/へ/を を確認して wa / e / wo 化し、チェックリストを再実行する |

### 運用ルール

- 修正は **1 回の再生成につき 1 箇所** (複数を同時に変えると、どの変更が効いたか分からなくなる)。
- 修正は必ず作品版側で行い、入稿版は再変換で作り直す (三原則 1)。
- 日本語曲は英語曲よりガチャ回数が嵩みやすい (仕上げまで数十生成という報告もある)。初回からテンポに合った歌詞量 (01 §4) を守ることが、回数を減らす最大の節約になる。
- モデル更新で日本語発音が改善しても、入稿変換を弱める判断は spec 更新 (update-spec) とセットで行う。**作品版は変換方針に関わらず不変** — これが二重管理の利点。
