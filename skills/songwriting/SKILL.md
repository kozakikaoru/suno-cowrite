# songwriting — 作詞ノウハウ参照資料集

このスキルはコマンドではなく、**作詞家 (lyricist) エージェントが執筆前に読む参照資料のコンテナ**です。マネージャーは作詞家を呼ぶとき、このディレクトリの絶対パスを呼び出しプロンプトに必ず含めます。作詞家は手癖で書かず、下の読み方ガイドに従って資料を読んでから書きます。

## 読み方ガイド

### 必読 (毎回)

| 資料 | いつ読む | 内容 |
|---|---|---|
| `references/01_song-structure.md` | 設計の前 | 曲構成理論、Suno メタタグ写像表、テンポ別歌詞量の目安 |
| `references/05_emotional-arc.md` | 設計の前 | 感情曲線の設計、1 番と 2 番の情報差、カタルシス設計 |
| `references/06_language-ja.md` | 日本語曲 | 日本語作詞の技術と Suno 入稿変換規則 (完全版) |
| `references/07_language-en.md` | 英語曲 | 英語作詞の落とし穴回避 (簡潔版) |
| `references/08_originality-checklist.md` | 納品の前 | オリジナリティ 3 観点検査と自己申告テンプレート |

06 / 07 は歌詞の言語で択一します (両言語が混在する曲は両方)。

### 曲の性格で選択

| 資料 | 読む条件 |

| `references/02_hook-design.md` | フック勝負の曲 (キャッチー狙い・シングル志向・ショート動画展開の想定)。迷ったら読む |
| `references/03_rhyme-and-rhythm.md` | 韻を効かせる曲、ラップ/語りセクションがある曲、譜割りを細かく制御したい曲 |
| `references/04_motif-and-lyric-devices.md` | モチーフの練りが要る曲 (世界観重視・物語性・バラード)、表現の既視感を避けたい曲 |
| `references/09_rhyme-advanced-ja.md` | 韻が主役の曲 (韻プロファイル heavy) で、**韻バンク (客演 rapper の素材) が渡されていないとき** (バンクが渡されていれば通読不要)。rapper (客演ラッパー) の主資料。ファイルが無ければ 03 のみで進めてよい |
| `references/10_hiphop-genre-ja.md` | ジャンルが HIPHOP/ラップ系の曲全般 (lyricist・rapper が読む。サウンド写像の節があるため hiphop 曲では composer にもパスが渡される)。ファイルが無ければ 03 のみで進めてよい |

## 作詞ワークフロー (5 段階、順番を崩さない)

### 1. 設計

ブリーフ (テーマ / 感情曲線 / 構成案 / コアモチーフ / NG) を読み、**1 行も書き始める前に**次を決めます。

- 構成表: セクション列とタグ列 (01 §3 の写像表)・各セクションの行数 (01 §5 のテンプレート)
- 音数割付: 行ごとのモーラ数 (英語はストレス数) の目安 (03 §4)
- 韻計画: どの行末を揃え、どこで外すか (03 §1。「効かせない」も計画のうち)
- モチーフ配置: コアモチーフをどのセクションでどう変奏するか (04 §2)
- 感情曲線: セクション × 強度の割付 (05 §1)
- 韻バンク: 客演 rapper の韻バンクが渡されていれば、韻計画の素材にする (候補の採否・改変は自由。最終著者は lyricist)

完了条件: 構成表が 1 枚にまとまっている。まだ歌詞は 1 行もない。

### 2. 執筆 (サビから書く)

- **サビ → Verse 1 → Pre-Chorus → Verse 2 → Bridge → 残り**の順で書きます。セクション順 (頭から) に書きません。
- サビが立たないうちは Verse に進みません。サビを書き終えたら 02 §7 のフック検査を通してから先へ進みます。

完了条件: 作品版の全セクションが埋まっている。

### 3. 推敲

- 音数の読み合わせ (03 §4 のモーラ計測)、感情曲線とのずれ (05 §7)、モチーフの回収漏れ (04 §8)、1 番と 2 番の情報差 (05 §3)、語尾の単調 (06 §4 / 07 §2) を検査します。
- 表現の修正はすべてこの段階の**作品版**で終わらせます。

完了条件: 読んだ各資料末尾の推敲チェックリストを通過している。

### 4. 入稿変換

- 作品版を凍結してから、06 第 2 部の規則表 (R1〜R9) で入稿版へ**機械的に**変換します。変換中に表現判断をしません。
- 変換で直したくなったら段階 3 に戻って作品版を直し、再変換します。入稿版を直接編集しません。
- 変換が不要な言語では「入稿版 = 作品版と同一」とし、その旨を明記します (07 §6)。

完了条件: 06 の変換チェックリストを全項目通過している。

### 5. オリジナリティ自己検査

- 08 の 3 観点 (フレーズ / 構造 / タイトル) で検査し、自己申告 (中心モチーフの出所 / 意図的に避けた既存曲 / 不安のある行) を書きます。
- 「不安のある行」はマネージャー経由でリサーチャーの Web 照合に回ります。迷ったら申告に載せます。

完了条件: 自己申告 3 項目が「検査を実施した上で」埋まっている (「なし」も判断の結果として明記)。

## この資料群の前提

- Suno の文字数上限・メタタグ語彙・モデル別挙動の最新値は **suno-spec (実効パスは呼び出しプロンプト参照) が正**です。本資料内の数値は執筆時の設計目安として扱います。
- 例文・サンプル歌詞はすべてこの資料のための自作です。実在曲の歌詞は一切引用していません (今後の改稿でも追加しません)。
- 修正依頼 (「2 番 A メロが早口」等) の対応手順は 06 第 3 部の症状別対処表を参照します。
