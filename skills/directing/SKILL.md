# directing — 演出ノウハウ参照資料集

このスキルはコマンドではなく、**演出家 (director) エージェントが設計前に読む参照資料のコンテナ**です。マネージャーは演出家を呼ぶとき、このディレクトリの絶対パスを呼び出しプロンプトに含めます。演出家は手癖で組まず、下の読み方ガイドに従って該当工程の資料を読んでから、世界観案・コンセプトブリーフ・物語管理を返します。

composer に対する composing、lyricist に対する songwriting と同じ位置づけの、director 専用ライブラリです。composer と違いジャンル別でなく**機能別 5 本**に分かれます。

## 読み方ガイド

references は毎回全読しません。**該当する工程のものだけ**を読みます。

| 資料 | いつ読む |
|---|---|
| `references/01_worldbuilding.md` | 世界観を新規に作る/深める前 (debut の世界観案、world.md 更新)。アーティスト像 (背景・立ち位置・語り口) を立てるときも |
| `references/02_concept-brief.md` | 新曲ブリーフのコアモチーフ・感情曲線を詰める前 (song)。迷ったら読む |
| `references/03_discography-narrative.md` | ディスコグラフィー物語管理・被り検知・次曲提案 (meeting、複数曲の位置づけ) |
| `references/04_visual-identity.md` | ビジュアルの方向を出すとき (debut のビジュアル方向、ジャケ/MV 方向、character-designer への申し送り)。**dual-audience** |
| `references/05_naming.md` | アーティスト名・シリーズ/連作名・EP/アルバム名・世界観固有語を決めるとき。**曲名は songwriting 11_title-craft が主管** (ポインタ) |

工程との対応の目安:

- **通常の新曲ブリーフ (song)** — 主に **02**。世界観に触れるなら **01**、物語上の位置づけを詰めるなら **03** を足す。ブリーフ 1 本に 5 冊全部は要らない
- **デビュー設計 (debut)** — **01** で世界観、**04** でビジュアル方向、**05** でアーティスト名候補
- **ディスコグラフィー会議 (meeting)** — 主に **03** (物語・被り・次の一手)。器の設計に **04** の視覚一貫性を足すこともある

該当工程の資料がまだ存在しなければ、存在する資料と director.md の運用ロジックだけで進めてよい。

## 運用規則

- **判定基準は常に world.md と artist.yaml** (整合判定は discography.md も)。資料は「どう設計・判定するか」の方法論であり、「このアーティストの世界観は何か」はアーティストファイルから読む (director.md の原則と接続)
- **04 は dual-audience**。character-designer に渡るのは §末「申し送り」節だけ。director は全節を読んでよい。director は視覚の**方向**を書き、英語生成プロンプトと具体的なキャラ造形は character-designer が書く (ここが境界)
- **05 の曲名**: 曲タイトルの型・表記・審査は **songwriting 11_title-craft (lyricist 主管)**。05 は曲より上位のネーミング (アーティスト名・シリーズ名・世界観固有語) を扱う。命名の道具立ては 11 と共有し、05 では重複させず「上位ネーミング固有の関心 (長期運用・横断一貫性・用語体系)」だけを持つ
- **02 の詳細実装**: 感情曲線・モチーフを歌詞の中でどう書くかは **songwriting 04/05 (lyricist 主管)**。02 は「ブリーフでどう指定するか (上位契約)」に限る
- **2 速度モデル**: references は年単位で古びない骨格だけを扱う。最新のビジュアル/命名トレンドはその場で書き換えず、**`:trend` 調査で補完**してブリーフ経由で反映する

## この資料群の前提

- 例 (世界観・モチーフ・アーティスト像・命名の類型など) はすべてこの資料のための自作です。実在アーティスト名・ブランド名・レーベル名は本文に書きません (今後の改稿でも追加しません)。固有名が要る場合は各資料の巻末「研究出典」欄のみに置きます
- 実在人物・実在ブランドに収束する記述をしない規則は、director と character-designer のオリジナリティ規則に従います
- Suno の仕様 (モデル・文字数上限・タグ語彙など) は **suno-spec (実効パスは呼び出しプロンプト参照) が正**。Style 方向は音楽の言葉で書けば足り、タグ確定・数値化は composer の仕事です
