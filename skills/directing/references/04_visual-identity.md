# 04 ビジュアルアイデンティティ — アートディレクションの上位契約

ビジュアルの方向を出すときに読む (debut のビジュアル方向、ジャケ/MV の方向づけ、character-designer への申し送り)。director が持つのは「方向づけ (日本語)」であり、英語生成プロンプトと個別キャラ造形 (髪型・顔立ち・服装) は書かない。それは character-designer の領分。この資料は dual-audience: **character-designer はこの資料を渡されたら §7「character-designer への申し送り」を必読** (余力があれば §3 色設計・§4 ジャケ方向も)。

> - 例 (トーン&マナー・色・モチーフの記述) はすべてこの資料のための自作。実在アーティスト・ブランド・レーベル名は本文に書かない。
> - 視覚オリジナリティ宣言: 特定の実在人物・既存作品のキャラクター・特定イラストレーターに収束する視覚記述をしない (character-designer.md の実在人物ガードと同一思想)。「〜風」が欲しいときは技法・年代・色彩・タッチに分解する。
> - 判定材料は world.md / artist.yaml / discography.md が正。この資料は「どう設計するか」の方法論だけを持つ。
> - 数値 (px・入稿仕様) は目安。配信入稿の実務値は suno-spec と各配信公式が正。最新のビジュアルトレンドはここに焼かず `:trend` で補ってブリーフに反映する。

## 1. ビジュアルアイデンティティは「システム」である — 視覚要素の体系

- ビジュアルアイデンティティは単一の資産 (ロゴ 1 枚) ではなく、色ルール・タイポ階層・画づくりの美学が一つの言語として反復されて初めて「認識」になる。
- 一貫性がデザインを言語に変える。反復する構図・繰り返す色アクセント・特定タイポのリズム・意図的な余白が、聴き手の中に「親しみ = 認識」を積み上げる。
- director が持つのは「方向 (規則の制定)」であり、実装 (英語プロンプト・character.md) は character-designer が持つ。この上下関係はコンセプトブリーフ (上位契約) と各専門 (実装) の視覚版であり、既存の分業とそのまま整合する。

視覚要素の体系 (何が一貫性を作るか) は次の 4 本柱に収斂する。**4 本ともルールを「制約」として設計する**のが要。

| 柱 | 役割 | durable な制約ルール |
|---|---|---|
| ロゴ・ワードマーク | アーティスト名の視覚的省略記号 | 凝ったシンボルより「様式化した文字」で十分。**40px でも読める**・単色でも成立する・流行より不変性を優先 |
| カラーパレット | 言語より速く伝わる感情のショートハンド | **2〜3 色に限定**し hex で記録 (主色 + 副アクセント + 中立地)。全 touchpoint で反復 |
| タイポグラフィ | 声・人格を運ぶ | **1〜2 書体** (見出し用 + 本文用)。商用ライセンスを確認し、小サイズ可読性を検証 |
| 写真・画づくりの美学 | プレス写真/SNS の「空気」を揃える | 同一の光の向き・同一の現像プリセット・撮影前に参照画像でブリーフ |

- 制約 = 一貫性 = 記憶。色を絞り、書体を絞り、リブランドを控えることが認識の蓄積になる。多色・多書体・頻繁なリブランドは、積み上げた認識をリセットする。
- 失敗パターン (裏返せばチェックリスト): 頻繁なリブランド (認識形成の中断) / モバイル視聴の軽視 (主要 touchpoint を落とす) / 過剰な複雑さ (一貫実行が不能になる) / 文書化の欠如 (協働で崩れる)。
- リブランドの時定数は 2〜3 年が目安。認識は時間をかけて積み上がるので、変えるなら era (§5) 単位で骨格を残して色・トーンだけ差し替える。
- アーティスト単位の「ミニ・ブランドガイド」(色 hex / 書体方針 / 画づくりトーン / ロゴ方針) を world.md 付近の固定資産として文書化し、複数のサブエージェント (character-designer 等) が同じ規則を参照できるようにする。

## 2. トーン&マナーの設計 — 世界観から視覚トーンを導く

- 視覚トーンは思いつきで決めず、world.md のテーマ核 (01) から導く。「この世界はどういう温度・質感か」を先に言語化し、それを視覚の空気に翻訳する。
- ムードボードの考え方: 「どう感じさせたいか」を先に決めてから、その感情方向に沿って参照画像を集める。見た目の綺麗さから集め始めない (綺麗さは目的ではなく結果)。
- トーン&マナーは色・光・質感・余白の総体であり、色 (§3) はその中で最も強く感情を運ぶ 1 要素。トーンを決めてから色を絞る順序を守る。

## 3. 色設計 — 感情から色を導く工程 (綺麗さから始めない)

- 色は「あとで決める飾り」ではなく「コンセプトから導く最初の設計項目」。ブリーフの段階で色まで踏み込む。
- 工程 (この順序を崩さない):
  1. 感情キーワードを先に決める (world.md のテーマ核・曲のコアモチーフから)。
  2. その感情方向でムードボードを作る (§2)。
  3. 2〜3 色のパレット (hex) に構築する。主色・アクセント・中立地の役割を割る。
- 温度の割り当て (目安): 暖色 = エネルギー/攻撃性、寒色 = 内省。系統対応の型 (自作例): 暗い緑・黒・くすみ系 = 内省的で陰のある作風、高コントラストの赤 × 黒 = 攻撃的でシャープな作風。
- 限定パレットは意図的で洗練して見え、多色は「焦点がない/混沌」に見える。ミニマルなパレットは物語と情感の余白を残す。
- 決めた hex は world.md / artist.yaml 付近に固定資産として書き留め、全曲・全 touchpoint で参照する。

## 4. ジャケット/カバーアートの方向づけ — サムネイル前提のチェックリスト

- 大前提: カバーは 95% がサムネイル (小サイズ) で消費される。**3000px で作り、100px に縮小して検証する**。読めない/濁るなら単純化する。
- ジャケ方向づけのチェックリスト (director → 画像生成ブリーフに載せ、character-designer に渡す):
  - [ ] 高コントラスト (明 × 暗)。低コントラストは小サイズで消える。
  - [ ] 単一の焦点 (一つの強い像・顔・グラフィック要素)。複雑なコラージュを避ける。
  - [ ] 文字はアーティスト名 + 曲名/アルバム名のみ。細字・スクリプト体は小画面で消えるので避ける。
  - [ ] カタログ横断で書体を 1〜2 種に統一する。
  - [ ] URL・SNS ハンドル・価格・explicit マークは入れない (プラットフォームが自動付与する)。
  - [ ] 入稿要件は 3000×3000px / 正方形 1:1 / RGB / JPG または PNG (細部数値は配信公式と suno-spec が正)。
  - [ ] 100px 検証で読めるかを最後に確認する。
- 人物あり vs モチーフのみ: どちらでも原理は同じ「単一の強い焦点」。強い写真 + 最小限のオーバーレイか、識別性の高いグラフィック (モチーフ) か、どちらかに寄せる。両者の中途半端な混在が最も弱い。
- アーティスト名 + 曲名の視覚扱いは、名前の表記設計 (05) と曲名の表記設計 (songwriting 11) と対で見る。
- ジャケの「視覚モチーフの方向 (何をどう見せたいか)」は director が出し、実際の英語プロンプト化と人物あり/モチーフのみの最終判断は character-designer (§7)。

## 5. ディスコグラフィー全体の視覚的一貫性 — 不変層/可変層の 2 層モデル

- 現代のリリースは単一ジャケでなく「アルバム期 (era)」= 視覚的宇宙として設計される。ジャケ・グッズ・MV・タイポ・SNS が独立せず、単一の美学を相互補強する。
- 一貫性と変化を両立させる 2 層モデル (最重要):
  - **不変層 (ブランド背骨)**: ロゴ・ワードマーク、基調書体、画づくりの根本トーン。era をまたいで保つ。
  - **可変層 (era 表現)**: 限定パレット、キービジュアルのモチーフ、写真の質感。アルバム期ごとに差し替える。
- この 2 層で「毎回新鮮なのに、同じアーティストと分かる」を実現する。**色は最も安全に動かせる可変層**であり、背骨を壊さず「新しい章」を宣言できる。
- 通し糸 (through-line): 各 MV・ポスター・ライブがアルバムの概念枠を延長し、全出力に一本の糸を通す。デザインは意味から導かれ、より大きな物語を貫く。
- ディスコグラフィーの色の変遷それ自体を、作品横断のナラティブ (章立て) として設計できる。物語管理は 03 (discography-narrative) と接続する。

## 6. バーチャルアーティストのビジュアル運用 (本プラグインの主用途)

顔を隠すこと (匿名・覆面・アバター) は装飾でなく成長エンジンであり、本プラグインが最も想定する運用。ここは厚めに扱う。

- なぜ「顔を隠す」ことがブランドになるのか — 4 つの複利効果:
  1. **好奇心 → 拡散**: 「この声は誰?」はフィードで抗いがたく、シェア・考察・ファン理論を生む。有料リーチ高騰時代の「無料の到達」。
  2. **作品 > 人物**: 顔出しを外すと、楽曲・映像・コンセプトが全てを背負う。人物駆動に疲れた層には「作品第一 = 誠実」に映る。
  3. **創作の自由と耐障害性**: 仮面の背後ではジャンル転換が反発を受けにくい。ボーカルが抜けても IP (アバター・ロア・様式) が個人より長生きし、カタログを脱リスク化する。
  4. **キャラ経済**: 強いアバターがグッズ・トランスメディア (マンガ/アニメ連携)・ライセンスを解錠する。ファンは曲でなく「世界」に参加する。
- **moat の実体**: 識別的な 音色 + ミックス + 色キー + アバターのシルエット + 物語モチーフ の総体。同等の品質と頻度で模倣するのが難しいほど堀は深い。world.md / character.md はこの moat を設計する場。
- バーチャルアーティストの 3 類型:

  | 類型 | 定義 | ブランドの要 |
  |---|---|---|
  | 完全フィクション型 | 人間の演者が不在。音源ソフト + キャラが本体 | キャラ + サウンドへの愛着。作家が入れ替わっても持続 |
  | アバター前面化型 | 背後に人間/チームがいるが、CG/イラストのキャラを前面に立てる | ビジュアルの一貫性 (髪色・シルエット等の署名) |
  | 声主導・匿名型 | 声を錨に、人物はシルエット/影で秘匿する | 「声」の一意性。ペルソナは可塑的でも声が不変 |

- キャラ設定 ↔ ブランディングの接続: アバターが全ブランディングの起点。まず種族・特徴・視覚モチーフを定義してキャラを識別可能にする。楽曲をロア (世界設定) を進める場として使い、director は「曲ごとにロアの何を進めるか」を管理する。
- 陳腐化の回避: 進化しない仮面は硬直する。対策は季節ごとの弧・アートディレクションの刷新・慎重に演出したロアの変化 (§5 の不変層/可変層がそのまま効く)。合成に振りすぎると「魂のなさ」に映るので、人間の趣味判断 + 合成の利便のハイブリッドを勝ち筋にする。
- director (世界観ビジュアル) と character-designer (キャラデザ実装) の関係: director が世界とトーンを決め、character-designer がアバターの外見を具体化する。受け渡しは §7。

## 7. character-designer への申し送り (必読)

director の視覚方向を character-designer が具体プロンプト + character.md に落とすときの受け渡し事項。**この節だけで実装に着手できるように書く**。

- **哲学**: director は方向 (日本語) を渡し、character-designer が英語生成プロンプトと外見の具体 (髪型・顔立ち・服装・外見アンカー) を作る。director は個別造形を指定せず、英語プロンプトを書かない。逆に character-designer は世界観・アーティスト像に触れる変更を独断せず director に確認する (character-designer.md の敬譲節と対をなす)。
- 受け渡す 4 点 (これを揃えて渡す):
  1. **トーン&マナー要約** — 1〜3 行。world.md (01) から導いた視覚の空気。
  2. **色** — 2〜3 色の hex と、主色/アクセント/中立の役割・温度の意図。
  3. **避ける視覚要素** — world.md の NG と、「特定の実在人物に収束する視覚記述をしない」指示。
  4. **世界観モチーフ** — 繰り返し出す視覚シンボル。ジャケ・キービジュアルで反復させる核。
- ジャケ依頼のときは追加で、対象曲のコアモチーフの「視覚翻訳の狙い (何をどう見せたいか)」を渡す。実際の英語プロンプト化と、人物あり/モチーフのみの判断は character-designer が行う。
- 実在人物ガードは両者が持つ。director 側は申し送りにも「特定の実在人物・既存キャラに収束する視覚記述をしない」を明記する。

## 8. 推敲チェックリスト

- [ ] 色は 2〜3 色に絞り hex で記録したか (多色で焦点を失っていないか)。
- [ ] 感情キーワード → 色温度 → パレットの順で導いたか (綺麗さから始めていないか)。
- [ ] ジャケ案は 100px で読めるか (高コントラスト・単一焦点・文字最小)。
- [ ] 入稿要件 (3000/1:1/RGB) と「自動付与情報を入れない」を満たすか。
- [ ] 不変層 (背骨) と可変層 (era) を分離して記述したか。
- [ ] バーチャルアクトなら moat (声 + 視覚システム) の核を言語化したか。
- [ ] §7 の 4 点 (トーン&マナー/色/避ける要素/モチーフ) を character-designer に渡せる形にしたか。
- [ ] 英語プロンプト・個別キャラ造形を書いていないか (それは character-designer の領分)。
- [ ] 実在人物・ブランド・レーベルに収束する視覚記述がないか。
